### API↔훅 (경계면 교차 비교)
일치. T10은 Route Handler/TanStack Query를 신설하지 않는 상세 경로(ADR-0003 서버 직접 호출)이므로, 검증 대상 경계면은 **page.tsx(생산자) ↔ filmography.tsx(소비자 `FilmographyEntry`) ↔ ContentCard(최종 소비자)** 이다.
- `FilmographyEntry`(filmography.tsx:26-39)는 filmography.tsx에서 export, page.tsx:33에서 `type` import — 단일 계약 원천, 이중 정의 없음.
- `toEntry`(page.tsx:53-74)가 산출하는 객체 `{key, href, title, posterPath, year, rating}` ↔ `FilmographyEntry` 필드/타입 정확히 일치.
- `FilmographyEntry` → `ContentCard`(content-card.tsx:23-34) props(`href/title/posterPath/year?/rating?`) 일치. `year: string|null`, `rating: number|null` 모두 ContentCard의 `?| null` 시그니처에 대입 가능.
- 유니온 판별 정확: `PersonCombinedCastCredit|CrewCredit`(types.ts:307-325)는 `media_type` 판별자 유니온. `toEntry`가 `media_type==="movie"`→`title/release_date`, else→`name/first_air_date`로 분기 → movie=`Movie&{}`(title/release_date 보유), tv=`TVShow&{}`(name/first_air_date 보유)와 정확히 매칭. snake_case(원본)→camelCase(뷰모델) 변환은 toEntry 한 곳에서만 수행돼 일관.
- `getPersonCombinedCredits` 반환 `PersonCombinedCredits.{cast,crew}`(client.ts:299-305 / types.ts:328-332)를 page.tsx:139-140이 각각 `normalizeCredits`로 소비 — 래핑/배열 기대 불일치 없음.

### 에러/엣지케이스 (01_ARCHITECTURE §4)
- 없는 id → notFound(): 준수. id 파싱 가드(비정수/음수 → notFound, page.tsx:127-129) + `getPerson` null → notFound(page.tsx:133-136). **순서 원칙 준수** — 인물 존재 확인(null→notFound)을 `getPersonCombinedCredits` 호출(page.tsx:138)보다 먼저 수행해, 없는 id가 credits throw로 error.tsx에 새지 않음(T7/T8 동일 패턴).
- fetch 실패 → error.tsx: 준수. `error.tsx` 세그먼트 경계 존재, `unstable_retry` 재시도 연결(error.tsx:19-34), `console.error(error)` 로깅(삼키지 않음). `unstable_retry` prop은 T7/T8에서 동일 사용·이미 QA PASS 승인된 이 코드베이스의 확립된 규약(03_qa_T8.md 명시).
- 429 패스스루: 준수. client.ts가 상태코드 보존·자체 재시도 없음(ADR-0004). 상세 경로는 Route Handler 없이 서버 직접 조회 → 429는 error.tsx 경계로 전파. 클라이언트 useQuery 미사용이므로 retry:1 정책 대상 아님.
- 데이터 결측(§2.9): 준수. biography 빈 문자열→"약력 정보가 없습니다."(page.tsx:193-195), profile_path null→이니셜 플레이스홀더(page.tsx:165-173), known_for_department/birthLine 결측→해당 Pill·줄 생략(page.tsx:182-189). 카드 개별 결측(poster/year/rating)은 ContentCard/PosterImage에 위임.
- 빈 리스트 섹션 숨김: 준수. cast/crew 모두 비면 `hasFilmography` 가드로 섹션 숨김(page.tsx:201-203). 한쪽만 비면 해당 토글 버튼 미렌더(filmography.tsx:114-129) + 컴포넌트 이중 가드(filmography.tsx:100-102).
- 최신순 정렬: 준수. `normalizeCredits`가 `media_type:id` 키로 중복 제거(Map, 첫 항목 유지) 후 개봉/방영일 내림차순 정렬(`b.date.localeCompare(a.date)`, ISO 사전식=시간순), 날짜 없는 항목은 명시 분기로 맨 뒤(page.tsx:92-118).

### 라우팅
일치. 필모 카드 `href`는 `toEntry`에서 movie→`/movie/{id}`, tv→`/tv/{id}`로 media_type 분기 산출(page.tsx:59/67). 두 동적 세그먼트 페이지 실재 확인(`src/app/movie/[id]`, `src/app/tv/[id]`). not-found.tsx의 `Link href="/"`(홈) 실재. `redirect`/`router.push` 사용 없음. 인물→작품 탐색 흐름(PRD §2) 완결.
