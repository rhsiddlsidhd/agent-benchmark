## T5 — 판정: PASS

검색 Route Handler + `useSearchInfinite` 훅(FR-2). 신규 3파일 경계면 교차 검증 완료.

### API↔훅 (일치)
- 생산자 `route.ts:48-49`: `searchMulti()` 결과를 그대로 `NextResponse.json(data)`. 반환 타입 `Paginated<MultiSearchResult>`.
- 소비자 `use-search-infinite.ts:43`: `const data: SearchResponse = await response.json()`, `fetchSearchPage(): Promise<SearchResponse>`.
- 공통 소스 `types.ts:10`: `SearchResponse = Paginated<MultiSearchResult>` — route/훅이 **동일 파일 참조**. shape 단일 진실원 확보.
- 빈 query 응답(`route.ts:31-36`)도 `SearchResponse`로 명시 타이핑되어 shape 유지.
- 런타임 실측(dev 서버 `/api/search?query=test`): top keys `page,results,total_pages,total_results`, `results[0].media_type="tv"` + `id` 존재 → 타입 선언과 정확히 일치. snake_case(TMDB 원본) 그대로 전달·소비로 변환 불일치 없음.

### 에러/엣지케이스 (§4 준수)
- 429/5xx 패스스루: `route.ts:51-56` `isTmdbError`로 좁힌 뒤 `error.status`가 400~599면 그대로, `status 0`(네트워크/타임아웃)은 502 매핑. 핸들러 자체 재시도 없음(ADR-0004). 예상외 에러는 `throw`로 Next 기본 500 위임(삼키지 않음).
- 재시도: 훅 `use-search-infinite.ts:61` `retry: 1`만 설정. 추가 재시도 로직 없음 — §4/ADR-0004 일치.
- 빈 query 이중 방어: 서버 `route.ts:30` 빈 문자열 → TMDB 미호출 + 빈 Paginated(200). 클라 `use-search-infinite.ts:60` `enabled: trimmedQuery.length > 0`. 런타임 실측 `query=` → `{"page":1,"results":[],"total_pages":0,"total_results":0}` HTTP 200 확인.
- getNextPageParam: `use-search-infinite.ts:59` `lastPage.page < lastPage.total_pages ? lastPage.page+1 : undefined`. 마지막 페이지(page==total_pages)에서 `undefined` → 조용히 정지(§4). off-by-one 없음. 빈결과(page 1, total_pages 0)도 `1<0`=false → `undefined`로 안전 정지.
- page 파라미터 정규화: `route.ts:41-43` `Number.isInteger && >=1`만 허용, NaN/0/음수/누락은 1로. 견고.
