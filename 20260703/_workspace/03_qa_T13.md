### 에러/엣지케이스 (준수)

- **useSyncExternalStore hydration 안전성(근거 타당)**: `getServerSnapshot` 이 항상 `false`(`:94`) → SSR·하이드레이션 첫 렌더가 서버와 일치, 커밋 후 `getSnapshot`(`:85`)이 URL 시드값으로 전환. 이는 문서화된 안전 패턴으로 hydration mismatch 를 만들지 않는다. useState+effect 회피 근거(React19 `react-hooks/set-state-in-effect` 린트 + 하이드레이션 불일치)는 타당하며, 실제 eslint EXIT 0 으로 회피 확인. getSnapshot 은 primitive boolean 을 반환해 referential stability 보장, lazy-seed 는 1회만 실행.
- **URL 양방향(레이아웃에서 useSearchParams 미사용)**: 읽기 = 마운트/전체 새로고침 시 `getSnapshot`→`readAdultFromUrl()`(`:57`)로 `window.location.search` 1회 시드. 쓰기 = `setStoreValue`→`syncAdultToUrl`→`history.replaceState`(`:66-76`, 얕은 갱신). `useSearchParams` 미사용이라 앱 전역 CSR bailout 을 유발하지 않아 정적 홈/상세 프리렌더 무영향(신규 코드 어디에도 useSearchParams 없음 — grep 확인). 근거 타당.
- **페이지 이동(검색↔디스커버) 상태 유지**: 스토어가 모듈 스코프(`:82`)라 Link 클라이언트 네비게이션 사이 메모리 유지 → includeAdult 가 queryKey 를 통해 목적지 페이지에서도 그대로 적용. 상태 유지 요구 충족. (URL 의 `adult` 파라미터 자체는 다음 토글 전까지 목적지 URL 에 즉시 반영되지 않을 수 있으나, 스토어가 쿼리의 단일 진실원이라 기능적 결손 아님. 새로고침·링크 공유 시나리오는 URL 시드로 정상 복원.)
- **discover buildQuery 병합 ↔ T12 URL 왕복 무충돌**: `buildQuery`(`discover-explorer.tsx:105-114`)가 fresh 생성 대신 `window.location.search` 기반 병합으로 변경. type/genres 만 set/delete 하고 `adult` 등 타 파라미터 보존. adult 측 `syncAdultToUrl` 은 반대로 adult 만 조작하고 type/genres 보존 — 두 replaceState 경로가 서로소 파라미터를 다뤄 대칭 보존. genre 토글 시 adult 유실 없음, 타입 전환(genres 비움) 시에도 adult 보존 확인. T12 초기 복원(server searchParams→initialType/initialGenreIds props)은 마운트 시점이고 buildQuery 는 핸들러에서만 호출되어 왕복 로직 충돌 없음.
- **role="switch"/aria-checked**: `adult-toggle.tsx:39-40` `role="switch"` + `aria-checked={includeAdult}` 정확 바인딩. 상태를 색상만이 아니라 도트 위치(justify-start/end) + 라벨 텍스트("숨김"/"표시", `:63`)로 병행 노출(색맹 대응, §6).
- **reduced-motion**: `useReducedMotion()`(`:34`) 분기로 `layout={!shouldReduceMotion}`(`:56`) — 감속 설정 시 layout 애니메이션 제거·즉시 스냅. §5 opacity 외 변형 제거 원칙 준수.
- **Provider 밖 사용 방어**: `useAdultContent`(`:147-155`)가 null 시 기본값 삼키지 않고 throw — 배선 누락 노출(에러 은닉 금지 준수).
