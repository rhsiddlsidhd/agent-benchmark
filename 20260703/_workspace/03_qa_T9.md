## T9 — 판정: PASS

TV 시즌/에피소드 모듈(FR-4). 경계면(route↔hook↔component) 교차 비교 + 라이브 end-to-end 실증 완료. 모든 항목 통과.

### API↔훅 (일치)
- 단일 소스: `route.ts`, `use-tv-season.ts` 모두 `@/src/features/tv-detail/types.ts`의 `SeasonResponse`(= `SeasonDetail`) 참조.
- 생산자: `route.ts:48` `getTvSeason()`(client.ts:289 → `Promise<SeasonDetail>`) 결과를 `NextResponse.json(data)`로 그대로 반환. 래핑 없음.
- 소비자: `use-tv-season.ts:30` `fetchSeason` → `Promise<SeasonResponse>`, `useQuery` 제네릭 추론 일치. `season-selector.tsx:69,75`가 읽는 `data.episodes`(→ `Episode[]`) 및 에피소드 필드(id/still_path/episode_number/name/air_date/overview) 전부 타입에 존재.
- 실패 shape: `SeasonErrorResponse = { error: string }` 400/패스스루에서 일관 사용. 훅은 HTTP status 기반으로 throw(에러 바디 파싱 안 함) — 정상.
- **라이브 실증**(`next start` + curl): `/api/tv/1399/season/1` → 200, `episodes[]` 실제 반환(응답에 `_id`/`episode_type` 등 추가 필드 있으나 구조적 타이핑 + 미참조로 무해).

### 에러/엣지케이스 (§4 준수)
- 429/5xx 패스스루: `route.ts:53-54` status 400~599 보존, status 0(네트워크/타임아웃) → 502. 핸들러 자체 재시도 없음. 훅 `retry: 1`(use-tv-season.ts:45), QueryProvider 기본도 `retry:1`. ADR-0004 부합.
- 라이브 실증: 비정수 id → 400, 없는 id(999999999) → 404 패스스루, Specials(season 0) → 200 허용. 전부 기대대로.
- notFound/error.tsx: `page.tsx:80-88` 없는 id/파싱 불가 → `notFound()`. 세그먼트 `src/app/tv/[id]/error.tsx` 존재.
- 결측 처리: 개요 없음 → "에피소드 개요가 없습니다."(season-selector.tsx:156), 스틸 null → BackdropImage 내부 ImagePlaceholder, 방영일 null → 표기 생략(formatAirDate→null 가드 :150).
- 빈 배열: 에피소드 [] → 목록 숨김·칩 유지(:69-71), 시즌 [] → SeasonSelector 섹션 미렌더(page.tsx:164).

### 라우팅 (일치)
- 신규 라우트 `/api/tv/[id]/season/[n]`, `/tv/[id]` 빌드 등록 확인(둘 다 ƒ Dynamic). 훅 fetch 경로 `/api/tv/${tvId}/season/${seasonNumber}`(:25)와 세그먼트 구조 정확히 매칭. person/tv 링크(page.tsx)는 T7/T8 기존 검증 범위.
