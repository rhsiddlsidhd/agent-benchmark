### API↔훅 (props/타입 흐름 end-to-end) — 일치

생산자(page.tsx)와 소비자(discover-explorer.tsx), 그리고 훅 계약을 동시 대조.

- **훅 제네릭→아이템 타입:** `useDiscoverInfinite<MediaType>({type, genreIds})` 호출(line 127). `type`이 유니온 변수 `MediaType`이라 조건부 타입 `DiscoverItem<"movie"|"tv">`가 분배되어 `Movie | TVShow`로 확장 → `data.pages[].results`가 `(Movie|TVShow)[]`. 소비 함수 `toCardData(item: Movie | TVShow)`(line 78)와 정확히 매칭. 제네릭 캐스팅으로 컴파일러가 못 잡는 shape 불일치 없음(tsc EXIT 0로 확정).
- **`"title" in item` 타입가드:** Movie(`title`)↔TVShow(`name`)를 `in` 연산자로 내로잉(line 79). `as`/`any` 단언 0건, `unknown` 불필요. Movie→`/movie/${id}`, TVShow→`/tv/${id}`, 날짜도 `release_date`/`first_air_date`로 타입별 정확 매핑(types.ts:66,117과 일치). CLAUDE.md 타입 안전 규칙 준수.
- **sentinel 연결:** `useIntersectionObserver({ enabled: hasNextPage && !isFetchingNextPage, onIntersect: fetchNextPage })`(line 130-135) — 훅 시그니처 정확 일치, 마지막 페이지/중복요청 정지 조건 올바름.

### URL searchParams 왕복 정합 — 일치(정밀 검증)

- **키 정합:** 쓰기 `buildQuery`가 `type`, `genres`(콤마) 사용(line 100-106) ↔ 읽기 `parseType(params.type)`, `parseGenreIds(params.genres, activeGenres)`(page.tsx:62-64). 파라미터 키 정확 일치.
- **`/api/discover` 호출 파라미터와 혼동 없음:** 훅 fetch는 `type/page/genreIds/includeAdult`(use-discover-infinite:57-64)로 별개 경계면. URL 동기화 키(`genres`)와 API 키(`genreIds`)가 달라도 서로 다른 계약이라 충돌 아님.
- **빈 선택 왕복:** genreIds 0개 → `genres` 파라미터 생략(`?type=movie`). 복원 시 `parseGenreIds(undefined,...)`→`""`→`[NaN]` 필터→`[]`. 무손실 왕복. ✔
- **오염/타 타입 방어:** `parseGenreIds`가 선택 타입의 실제 장르 목록(`activeGenres`)에 존재하는 양의 정수만 통과(Number.isInteger + Set.has). 링크 조작/타 타입 id 유입 차단. ✔
- **새로고침 복원:** `useState(initialType)`, `useState(initialGenreIds)`로 초기값 주입(line 114-115) → 서버 파싱값이 그대로 초기 상태. ✔
- **타입 전환 처리:** `handleSelectType`가 타입 변경 시 선택 비우고 URL 동기화(line 154-163). 타입별 장르 id 상이 문제 회피. 같은 타입 재선택 no-op(단일 선택 강제).

### 접근성(미디어타입 단일선택 FilterChip 재사용) — 문제 없음(권고 1건)

- 미디어 타입 토글은 `role="group" aria-label="미디어 타입"` 안에서 FilterChip(`aria-pressed`, 체크 아이콘 병행) 2개로 구성(line 193-208). aria-pressed 토글 버튼 그룹은 WAI-ARIA에서 상호배타 단일선택의 **허용된 대안 패턴**이며 상태를 색상만으로 구분하지 않아 §6/색맹 대응 준수. T11에서 PASS된 FilterChip 계약을 그대로 재사용해 일관적. **위반 아님.**
- (권고, 비차단) 상호배타 단일선택의 가장 정밀한 시맨틱은 `role="radiogroup"`+`role="radio"`/`aria-checked`다. 향후 개선 여지는 있으나 T12 판정에 영향 없음.
- 그 외: 페이지 h1 1개, GenreFilter `role="group"`, sr-only `aria-live="polite"` status로 로딩/무결과/N개/에러 안내(line 170-222). ✔
