### soft-404 재검증 (implementer 주장 직접 검증)
`node_modules/next/dist/docs/01-app/03-api-reference/03-file-conventions/loading.md` "Status Codes" 절 원문 확인:
> "The response body starts streaming when a Suspense fallback renders (for example, a `loading.tsx`)... it is not possible to change the status code after streaming started."
> "a `<meta name="robots" content="noindex">` tag... prevents search engines from indexing that URL even if the HTTP status is 200."

즉 soft-404(200+noindex)는 **loading.tsx 등으로 스트리밍이 시작된 이후에만** 발생하는 Next 16 표준 동작이며, 스트리밍이 시작되지 않으면 `notFound()`가 실제 404를 반환한다 — implementer 주장과 일치.

**단, implementer의 "회귀 없음" 검증 방법은 결함이 있었다.** `02_implementer_T14.md:60`은 "`movie/[id]/loading.tsx` 제거 상태에서도 동일하게 200"이라고만 확인했는데, 이때 **루트 `src/app/loading.tsx`(역시 T14 신규 파일)는 그대로 남아 있었다.** 루트 loading.tsx는 루트 레이아웃 자식 전체(모든 하위 라우트 포함)에 대해 Suspense 경계를 형성하므로, movie 세그먼트 loading.tsx를 지워도 루트 loading.tsx가 스트리밍을 이미 트리거해 200이 나오는 게 당연하다 — 즉 부분 제거만으로는 회귀 여부를 판별할 수 없는 테스트였다.

직접 재검증(양쪽 loading.tsx 모두 제거 후 원복 — 최종 git status 클린 확인):
- **T14 loading.tsx 포함(현재) 상태**: `/movie/999999999` → `200`(soft-404).
- **루트+세그먼트 loading.tsx 모두 제거(T14 이전 상태 재현)**: `/movie/999999999` → `404`, `<h1>`도 실제 DOM에 렌더(스트리밍 없이 즉시 완성 응답).

**결론: T14의 loading.tsx 추가가 실제로 `/movie(tv/person)/[id]`의 잘못된 id 응답을 하드 404 → soft-404(200+noindex)로 바꾼 것이 맞다.** 이는 회귀가 아니라 "T7/T8/T10 시점부터 이미 이랬다"는 `02_implementer_T14.md:60`의 서술이 부정확하다는 뜻이다.

이것이 실제 결함(REDO/FIX 대상)인지는 별도 판단 필요:
- `01_ARCHITECTURE.md §4`는 "TMDB 404 응답 시 `notFound()` 호출 → App Router 표준 `not-found.tsx`"만 요구하고 **HTTP 상태 코드 자체는 명시하지 않는다.** 실제로 `notFound()`가 호출되고 not-found.tsx가 렌더되며, `noindex`로 검색엔진 색인은 차단된다 — 아키텍처 정책·SEO 요구(NFR-1 Lighthouse SEO 80+) 모두 충족.
- Next 공식 문서도 이 트레이드오프(로딩 스켈레톤을 위해 세그먼트를 스트리밍하면 상태 코드를 이후에 바꿀 수 없음)를 표준 동작으로 문서화하고 있고, "compliance/analytics상 진짜 404가 필요하면 스트리밍 시작 전에 존재 여부를 확인하라"고 대안을 제시하지만 이는 loading.tsx(CLS 방지, NFR-1/3)를 포기해야 하는 트레이드오프다.
- 따라서 **코드 수정은 불필요**하다고 판단한다(REDO/FIX 대상 아님). 다만 `02_implementer_T14.md`의 근거 문장(60번 줄)이 사실과 다르므로 **기록 정정만 권고**한다 — "T14 이전에도 이 동작이었다"가 아니라 "T14가 이 동작을 유발했지만 아키텍처 정책·SEO 요구를 여전히 만족하는 의도된 트레이드오프"로 수정.
