## 1. 배경 (Background)

TMDB(The Movie Database)는 방대한 영화·TV 데이터를 보유하고 있으나, 공식 사이트나 기존 서비스들은 콘텐츠 → 출연진/감독 → 다른 작품으로 이어지는 탐색 흐름이 매끄럽지 않다. 본 프로젝트는 실사용 서비스가 아닌 **학습/포트폴리오 목적**으로, TMDB API를 활용해 직관적인 탐색형 웹 인터페이스를 구축한다.

## 2. 목표 (Goals)

- TMDB API 기반으로 영화/TV 콘텐츠를 탐색할 수 있는 웹 인터페이스를 제공한다
- 콘텐츠 → 출연진/감독 → 다른 작품으로 끊김 없이 이어지는 탐색 경험을 구현한다
- 반응형 + 성능 최적화된 SPA를 구축해 포트폴리오로 활용한다

**Non-Goals**
- 회원가입/로그인, 사용자 인증
- 찜하기/리뷰 작성/평점 부여 등 개인화 기능
- 별도 다국어 i18n 시스템 구축 (TMDB API의 language 파라미터 고정값만 사용)

## 3. 대상 사용자 (Target Users)

**페르소나: 콘텐츠 발견러**
영화/드라마를 좋아하며, 특정 작품을 보다가 마음에 든 배우·감독의 다른 작품까지 자연스럽게 이어서 탐색하고 싶어하는 일반 사용자. 회원가입이나 개인화 기능보다 빠르고 직관적인 탐색 경험을 중요시한다.

## 4. 요구사항 (Requirements)

### 4.1 기능 요구사항 (Functional)

| ID | 요구사항 | 우선순위 | 비고 |
|----|----------|----------|------|
| FR-1 | 홈 화면 — 인기/트렌딩 영화·TV 노출 | Must | |
| FR-2 | 통합 검색 (영화/TV/인물, TMDB multi-search) | Must | 결과는 타입별 섹션 구분 |
| FR-3 | 영화 상세 페이지 (줄거리, 출연진, 감독 등) | Must | |
| FR-4 | TV 상세 페이지 (개요 + 시즌/에피소드 목록) | Must | 시즌/에피소드 목록까지 포함 |
| FR-5 | 인물(배우/감독) 상세 페이지 (필모그래피) | Must | |
| FR-6 | 장르/필터별 탐색 | Must | |
| FR-7 | 성인 콘텐츠 노출 토글 (include_adult) | Must | |

### 4.2 비기능 요구사항 (Non-Functional)

| ID | 요구사항 | 기준 |
|----|----------|------|
| NFR-1 | 성능 | 이미지 lazy loading, 리스트 무한스크롤/페이지네이션, 동일 API 응답 캐싱 적용. Lighthouse Performance 90점 이상 |
| NFR-2 | 보안 | TMDB API 키를 클라이언트 번들/저장소에 노출하지 않음 |
| NFR-3 | 반응형/UX | 모바일·데스크톱 반응형 레이아웃, API 로딩/에러 상태를 명확히 표시 |
| NFR-4 | 접근성/품질 | Lighthouse Accessibility 90+, Best Practices 90+, SEO 80+ |

## 5. 사용자 스토리 / 유스케이스

- As a 콘텐츠 발견러, I want 홈 화면에서 인기/트렌딩 영화·TV를 바로 확인하고 싶다, so that 뭘 볼지 빠르게 발견할 수 있다.
- As a 콘텐츠 발견러, I want 제목으로 영화/TV/인물을 통합 검색하고 싶다, so that 원하는 콘텐츠를 빠르게 찾을 수 있다.
- As a 콘텐츠 발견러, I want 영화 상세 페이지에서 출연진/감독 정보를 보고 싶다, so that 관심있는 인물의 다른 작품도 이어서 탐색할 수 있다.
- As a 콘텐츠 발견러, I want TV 상세 페이지에서 시즌/에피소드 목록까지 확인하고 싶다, so that 시청 전 시리즈 구성을 파악할 수 있다.
- As a 콘텐츠 발견러, I want 배우/감독 상세 페이지에서 필모그래피를 보고 싶다, so that 좋아하는 인물의 작품을 이어서 탐색할 수 있다.
- As a 콘텐츠 발견러, I want 장르별로 콘텐츠를 필터링하고 싶다, so that 취향에 맞는 콘텐츠를 좁혀서 찾을 수 있다.
- As a 콘텐츠 발견러, I want 성인 콘텐츠 노출 여부를 토글로 제어하고 싶다, so that 원치 않는 콘텐츠를 걸러낼 수 있다.

## 6. 성공 지표 (Success Metrics)

Definition of Done 방식 (실사용자 없어 전통적 KPI 대신 적용):
- 필수 화면(홈/검색/영화상세/TV상세/인물상세) 전부 동작
- 콘텐츠 → 인물 → 다른 작품 탐색 흐름이 끊김 없이 연결
- 반응형 레이아웃 적용 및 배포 완료
- Lighthouse: Performance 90+, Accessibility 90+, Best Practices 90+, SEO 80+

## 7. 제약사항 (Constraints)

- 1인 개발, 명시적 마감 기한 없음 (자율 페이스)
- TMDB API 무료 티어 rate limit 준수
- 기술 스택(프레임워크/언어 등)은 [01_ARCHITECTURE.md](./01_ARCHITECTURE.md)에서 결정 — PRD 범위 아님
