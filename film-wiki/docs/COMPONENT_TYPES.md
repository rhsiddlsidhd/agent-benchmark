# COMPONENT_TYPES

> 컴포넌트 Props/타입 전용 컨벤션. 이 문서는 컴포넌트 타입 규칙만 다룬다 — 네이밍/구조 등 다른 컨벤션은 각 CLAUDE.md를 따른다.

## Props

- 컴포넌트 Props를 다른 파일에서 참조할 때 인터페이스를 복제·재정의하지 않는다 — `ComponentProps<typeof Component>`로 파생한다.
- Props 인터페이스를 컴포넌트 파일 밖으로 미리 분리해두지 않는다 — 렌더링 없이 shape 자체를 값으로 참조해야 하는 경우(예: 여러 컴포넌트가 구조적으로 동일한 props shape 공유, 테스트/스토리 fixture)에만 분리한다.
- 분리할 때 소유 컴포넌트명을 그대로 붙이지 않는다 — 역할·형태 명사로 개명한다(예: `ButtonProps`처럼 원 소유 컴포넌트에 묶인 이름 그대로 승격하지 않는다).
- 분리된 Props 타입은 `src/types/`에 둔다 — 폴더가 없으면 그 시점에 새로 만든다. 쓰이지 않는 빈 폴더를 미리 만들어두지 않는다.
