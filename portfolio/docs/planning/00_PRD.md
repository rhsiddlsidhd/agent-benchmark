## 1. 배경 (Background)

테니스 관련 특정 주제에 대한 대중 관심도/화제성을 정량적으로 측정할 도구가 없다. 검색량 추세(네이버 데이터랩)와 특정 커뮤니티(테니스피플) 게시글 수 추이를 결합해 화제성을 판단하고자 `portfolio`(검색량+게시글수로 관심도/화제성 측정)를 기획.

## 2. 목표 (Goals)

- 특정 주제 검색량 추세(네이버 데이터랩 상대지수 0~100, 일/주/월 단위)를 차트로 시각화
- 테니스피플(tennispeople.kr, bbs_19 게시판) 게시글 수 추이를 차트로 시각화
- 기간(6개월/1년 등) 지정 게시글 date+title 기반 LLM 요약 제공

**Non-Goals**
- Google Trends API 연동 (공식이나 현재 승인 대기 중, 사용 불가)
- 비공식 라이브러리(pytrends) 사용
- 게시글 본문 수집/분석 (목록 메타데이터만 사용, 제목 기반 빈도/추세 분석이 목적)
- 웹사이트 방문자수 측정 — "트래픽"은 검색량 추세만 의미

## 4. 요구사항 (Requirements)

### 4.1 기능 요구사항 (Functional)

| ID | 요구사항 | 우선순위 | 비고 |
|----|----------|----------|------|
| FR-1 | 네이버 데이터랩 검색어트렌드 조회 및 시각화 (일/주/월 단위, 상대지수 0~100) | | |
| FR-2 | 테니스피플 bbs_19 게시판 게시글 메타데이터(idxno/제목/작성자/날짜/조회수) 수집·저장 | | 본문은 수집 제외 |
| FR-3 | 기간별(6개월/1년 등) 게시글 수 추이 및 상대 구간 필터링 | | 분석 시점(`new Date()`) 기준 |
| FR-4 | 기간 지정 게시글 date+title 기반 LLM 요약 제공 (OpenAI API) | | |

### 4.2 비기능 요구사항 (Non-Functional)

| ID | 요구사항 | 기준 |
|----|----------|------|
| NFR-1 | 상시 서버 비용 최소화 | Vercel 서버리스 함수 + 무료 티어 우선, FastAPI/Render 상시 서버 미사용 |
| NFR-2 | 크롤러 중복 방지 | idxno PK + UPSERT, 최초 1회 풀 크롤링 후 증분 크롤링 |
| NFR-3 | secret 노출 방지 | 네이버 client_secret, OpenAI API 키는 서버리스 함수 경유로만 접근 |

## 5. 사용자 스토리 / 유스케이스

- As a 테니스 관심 사용자, I want 특정 주제 검색량과 게시글 수 추이를 한 화면에서 보고 싶다, so that 화제성을 직관적으로 파악할 수 있다.
- As a 사용자, I want 기간별 게시글 내용을 요약해서 보고 싶다, so that 게시글을 일일이 읽지 않아도 흐름을 파악할 수 있다.

## 7. 제약사항 (Constraints)

- 트래픽(검색량) 데이터소스는 네이버 데이터랩 단독 사용 (Google Trends API 승인 전까지)
- 테니스피플 robots.txt 없음(차단 규칙 없음)이나 요청당 1~2초 딜레이 준수, 약 366페이지 순회
- 예상 저장 용량 1MB 미만 (SQLite/CSV 검토 기준, 최종 저장소는 Supabase)
